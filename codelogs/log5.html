Code log 5: Images

  1. What are the differences between presenting an image using an HTML tag (like ) and presenting it as the background of an HTML element (using the CSS property background-image) in terms of differences in code as well as differences in functionality and layout.

    The difference between presenting an image using HTML tag of and presenting it as a background image using CSS property is that for HTML tag the image is placed directly on the web page like having the image embedded into the page to make it as part of the content presented. On the other hand an image using CSS property is used for decorative purposes like placing a specific image behind HTML elements or other background properties.

  2. Two other CSS properties that affects the display of background images are:
    • Background size: Which determines the size for the background image based on four ways using the syntax keywords of auto, cover and contain. You can have one value to give the background image width with auto height or two values to give it a specific height in addition to width. Contain allows the background image to resize to fit the entire element.
      1. Background size: 50px; means image 50px wide and auto adjusted height
      2. Background size: 50px, 70px; means image 50px wide and 70px for height
    • Background position: Determines where to place the background image in an element such as the top, bottom, left and right, or center. Two words are used to specify placement, and one word is used for simple positioning.
  3. Defining the differences between dimensions and resolution in a bitmap image:

    The difference between dimensions and resolutions in a bitmap image are that dimensions determine the width and height of the bitmap in pixels while resolution determines how clear the detail in an image is based on density of the pixels.

  4. Extra Cedit: What are the similarities and differences between these image file formats? Why are some better than others for specific uses?
    • GIF: Used for simple and small images and animations.
    • PNG: Used for higher image quality to support transparency but can be a large file size. It is often used when images need to be precise and clear as possible. Commonly used for logos and graphics with transparent backgrounds.
    • JPG/JPEG: Used commonly for photos or still images but does not offer transparency like PNG and while it has a lower file size than PNG it can lower the quality of the image.
    • WebP: Used for creating smaller file sizes while maintaining high quality of image or animation. It supports transparency and offers better compression than PNG or JPEG.
    • SVG: Based on vector images and used for interface elements.
